Seguridad Nativa: El Bloque `secure`

Iridium implementa el paradigma "Secure-by-Default". Entendemos que en el Backend y la IA, la protección de datos no debe ser opcional, sino una instrucción del núcleo del lenguaje.

1. El Bloque `secure:`

Cualquier operación de guardado, log o impresión dentro de este bloque activa automáticamente capas de protección y encriptación de datos en tránsito.

secure:
    var password = "clave_secreta_77"
    db.save(password) // Encriptación automática en la DB
    out "Guardando: {password}" // Imprime ********
end

2. Control de Visualización Manual

Si necesitas depurar una variable sensible, puedes forzar su visualización desactivando el cifrado de salida mediante metadatos en la interpolación.

out "{password, encrypted: false}" // Muestra el valor real

3. Aislamiento de Secretos

Las variables declaradas dentro de un bloque secure tienen un tiempo de vida restringido y se limpian de la memoria volátil (RAM) inmediatamente después de que el puntero de ejecución sale del bloque, previniendo ataques de lectura de memoria.

Protección Anti-Leak

Incluso si un atacante logra acceso a los logs del servidor, cualquier salida generada bajo el modo secure estará ofuscada por defecto, reduciendo el riesgo de exposición de credenciales.
