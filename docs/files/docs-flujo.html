Control de Flujo

Iridium utiliza una combinación de indentación y palabras clave de cierre para garantizar que el flujo lógico sea legible y fácil de depurar en entornos de Backend.

1. Condicionales: `if / else`

La estructura básica utiliza : para abrir el bloque. No es obligatorio el uso de paréntesis, pero se recomienda para mejorar la claridad en condiciones complejas.

if (precision > UMBRAL):
    out "Modelo estable con {precision}%"
else:
    out "Reajustando pesos..."

2. Bucle For (Estilo C/Java)

Para iteraciones numéricas precisas, Iridium adopta la sintaxis clásica de tres expresiones. Es ideal para controlar épocas de entrenamiento en IA.

for (var i = 0; i < 100; i++):
    out "Procesando época: {i}"

3. Bucle While y Cierre `end`

El bucle while se ejecuta mientras la condición sea verdadera. Todos los bloques extensos en Iridium pueden cerrarse con end para evitar ambigüedades en archivos grandes.

while (energia > 0):
    energia -= 1.5
    out "Energía restante: {energia}"
end

4. Selección: `switch`

Evita múltiples if anidados utilizando la estructura de selección nativa.

switch (estado):
    case "activo": conectar()
    case "error": out "Fallo detectado"
    default: out "Estado desconocido"
end
