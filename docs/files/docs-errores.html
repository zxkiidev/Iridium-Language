Errores Comunes y Soluciones

Esta página recoge los errores más frecuentes al desarrollar con Iridium, sus causas y cómo resolverlos. Los errores del motor siempre tienen el prefijo Ir-Error para distinguirlos de errores del sistema operativo.

Ir-Error: Inmutable violation

Causa: Intentaste reasignar una variable declarada con const.

const TASA = 0.01
TASA = 0.02 // ← Error fatal: constante protegida

✅ Solución: Usa var si el valor debe cambiar, o mantén la constante si es un hiperparámetro fijo del sistema.

Ir-Error: Archivo no encontrado

Causa: El motor no puede localizar el archivo .iri o el .env.

[Ir-Error] Archivo no encontrado: main.iri

✅ Solución: Asegúrate de ejecutar iridium main.iri desde el directorio donde está el archivo. Para el .env, debe estar en la misma carpeta que tu script.

[DB] Llamá db.config() o db.sqlite() primero

Causa: Se intentó ejecutar una consulta antes de configurar la base de datos.

// ❌ Sin configurar primero
var filas = db.query("SELECT * FROM users")

✅ Solución: Llama a db.config("host", "user", "pass", "db") para MySQL o a db.sqlite("archivo.db") al inicio del script, antes de cualquier consulta.

iridium no se reconoce como un comando

Causa: El PATH del sistema no fue actualizado tras la instalación, o la terminal no fue reiniciada.

✅ Solución: Cierra y vuelve a abrir la terminal (CMD / PowerShell). Si persiste, verifica manualmente que C:\Program Files\Iridium esté en tus variables de entorno del sistema.

Ruta no encontrada: GET /ruta

Causa: Se hizo una petición HTTP a una ruta que no fue definida con route, o se usó el método HTTP incorrecto.

// Definida como POST, pero se llama con GET
route "/datos" => "POST":
    ...
end

✅ Solución: Verifica que el método HTTP del cliente coincida con el declarado en el route. Puedes aceptar múltiples métodos separando con coma: => "GET", "POST".

Variable muestra {Indefinido} en out

Causa: Se intentó interpolar una variable en out que no estaba definida en ese scope, o cuyo nombre tiene un error tipográfico.

var nombre = "Iridium"
out "Hola {nombe}" // ← typo: 'nombe' no existe

✅ Solución: Verifica la ortografía exacta del nombre de la variable. Usa la extensión de VS Code para resaltar variables no declaradas.

¿Encontraste un error nuevo?

Abre un Issue en el repositorio oficial de GitHub describiendo el error, el código que lo reproduce y el mensaje exacto del motor. La comunidad y el equipo de Iridium responden activamente.
