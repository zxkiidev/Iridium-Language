Conectividad de Datos Nativa

Iridium simplifica la persistencia de datos mediante el bloque database. El motor gestiona internamente el pool de conexiones y la seguridad de las credenciales.

1. Conexión Segura

Combinando el bloque secure y env, puedes conectar a una base de datos sin exponer jamás las credenciales en el código fuente.

secure:
    database myDB = connect("postgresql", host: FROM env.file)
    // La conexión está protegida y cifrada
end

2. Consultas Híbridas

Iridium permite realizar consultas utilizando una sintaxis limpia que mezcla SQL con la lógica del lenguaje. Los resultados se mapean automáticamente a colecciones de Iridium.

var usuarios = myDB.query("SELECT name FROM users WHERE active = true")

for (user in usuarios):
    out "Usuario activo: {user.name}"

3. El Operador Sponge en DB

Es muy común que una consulta falle o no devuelva resultados. Aquí es donde el operador ~> brilla, permitiendo definir un dataset de respaldo.

// Si la DB está caída o vacía, usa la lista local
var data = myDB.fetchAll() ~> {"Invitado", "Anónimo"}

Motores Soportados

En su versión V1, Iridium soporta nativamente PostgreSQL, MySQL y SQLite. El soporte para bases de datos vectoriales (esenciales para IA) está en desarrollo para la V2.
