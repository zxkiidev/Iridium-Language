Roadmap: V2 y V3

Iridium V1 es el punto de partida: un motor interpretado en Python que ya permite construir APIs completas, trabajar con bases de datos y ejecutar operaciones de ciencia de datos. Pero esto es solo el comienzo.

ACTUAL

V1 — Intérprete Nativo (Python)

El motor actual traduce y ejecuta código .iri en tiempo real sobre Python. Ofrece la experiencia completa del lenguaje sin necesidad de compilación.

  • ✅ Tipado híbrido (dinámico y estricto)
  • ✅ Operador Sponge (~>), I/O, colecciones y rangos
  • ✅ Control de flujo: if/else, while, for, switch
  • ✅ Clases y objetos con métodos
  • ✅ Sistema de rutas HTTP (route + serve)
  • ✅ Base de datos nativa: MySQL y SQLite
  • ✅ Bloque secure y env
  • ✅ Motor de tensores básico y objeto ia
  • ✅ Extensión oficial para VS Code
EN DESARROLLO

V2 — Compilación a Bytecode

La V2 elimina la dependencia de Python en producción. El código .iri se compilará a un bytecode propio que ejecutará una máquina virtual (VM) escrita en C, logrando un salto de rendimiento significativo.

  • 🔵 Compilador iricc: .iri → .iribc (bytecode Iridium)
  • 🔵 Stack Machine propia — sin dependencia de Python
  • 🔵 Gestor de paquetes: iripkg install nombre
  • 🔵 Soporte nativo para bases de datos vectoriales (para IA)
  • 🔵 Modelos de IA propios: archivos .irimodel
  • 🔵 Módulos importables: import mi_modulo
  • 🔵 Soporte multi-archivo de proyecto
  • 🔵 Depurador interactivo con iridium debug
VISIÓN FUTURA

V3 — Compilación Nativa (LLVM)

La V3 representa la visión final de Iridium: un lenguaje compilado a código máquina nativo mediante LLVM, con gestión de memoria manual controlada y capacidad de entrenamiento distribuido de IA sin frameworks externos.

  • 🟣 Backend LLVM: compilación a binario nativo (.exe, ELF)
  • 🟣 Punteros seguros (sin segfaults involuntarios)
  • 🟣 Gestión manual de memoria con sintaxis alloc / free
  • 🟣 Entrenamiento distribuido multi-nodo desde el lenguaje
  • 🟣 Soporte GPU nativo (CUDA/ROCm) sin librerías externas
  • 🟣 Interoperabilidad con C/C++ mediante extern
  • 🟣 Compilación cruzada para ARM y WASM
