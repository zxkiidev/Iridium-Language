Manejo de Red Nativo

Iridium trata las peticiones HTTP como ciudadanos de primera clase. Puedes definir la infraestructura de tu API directamente en la gramática del código, facilitando el despliegue de microservicios de IA.

1. Definición de Endpoints

Usa los verbos HTTP (GET, POST, PUT, DELETE) seguidos de la ruta para crear un punto de acceso. Iridium mapea automáticamente los parámetros de entrada.

POST, GET "/analizar" => (data: tensor, model_id: int):
    if (method == "GET"):
        return {status: 200, msg: "Servidor Ir activo"}

    // Lógica de predicción
    var res = ia.predict(data, model_id)
    return {resultado: res}

2. El comando `serve`

Activa el servidor con una sola instrucción. Iridium gestiona internamente el multi-threading para soportar múltiples conexiones simultáneas.

serve(port: PORT, host: "localhost")

3. Integración con Variables de Entorno

Es una práctica recomendada combinar el manejo de red con el bloque env para mantener la resiliencia del puerto y el host.

env of ".env":
    const PORT: int = 8080 FROM env.file

serve(port: PORT)
Estructura Requerida

Para que env.file funcione, el archivo .env debe estar en la misma ruta (directorio) que tu script .iri:

/proyecto
├── main.iri
└── .env

Microservicios Listos

Iridium está diseñado para ser "Docker-Friendly". Al compilar tu script .iri, el servidor nativo se levanta con una huella de memoria mínima, ideal para contenedores en la nube.
