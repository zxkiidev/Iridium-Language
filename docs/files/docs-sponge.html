Operador Sponge (~>)

El operador Sponge (Esponja) es una herramienta de resiliencia nativa en Iridium. Su función es "absorber" posibles fallos en operaciones dinámicas, asignando un valor por defecto si la instrucción principal falla.

1. Concepto de Resiliencia

En el desarrollo de Backend e IA, a menudo dependemos de datos externos que pueden ser nulos o erróneos. En lugar de usar bloques try-catch complejos, Iridium propone una sintaxis lineal:

// Si la API falla, se asigna el valor por defecto "Localhost"
var host = api.getHost() ~> "127.0.0.1"

2. Uso con Cálculos Matemáticos

El operador Sponge es ideal para prevenir errores de división por cero o resultados indefinidos en modelos matemáticos:

var ratio = (puntos / total) ~> 0.0
out "Ratio calculado: {ratio}"

3. Diferencia con la Asignación Estándar

Operador Comportamiento ante error
= Lanza una excepción y detiene el hilo.
~> Captura el error y usa el valor de la derecha.

Nota Técnica

El operador Sponge es especialmente potente cuando se trabaja con la carga de archivos env. Si una variable de entorno no existe, tu servidor Iridium puede seguir funcionando con una configuración de respaldo.
