Tipado Híbrido

Iridium ofrece un sistema de Tipado Híbrido. Esto significa que puedes elegir entre la agilidad del tipado dinámico para prototipos rápidos o la seguridad del tipado estricto para sistemas de IA en producción.

1. Variables Dinámicas (`var`)

Ideal para cuando el tipo de dato puede cambiar o no es crítico para el rendimiento inmediato.

var nombre = "Zadquiel" // El motor infiere que es string
var datos = 100
datos = "Cien" // Permitido en modo dinámico

2. Tipado Estricto

Al definir el tipo, Iridium optimiza la memoria. Según nuestras pruebas, esto puede aumentar el rendimiento hasta un 500% en operaciones matemáticas complejas.

var precision: float = 0.98
var epoca: int = 10

3. Constantes Inmutables (`const`)

Las constantes no pueden ser reasignadas una vez definidas. Son esenciales para hiperparámetros de modelos de IA.

const UMBRAL = 0.85
const ID_NUCLEO: int = 77

Error de Compilación: Si intentas hacer UMBRAL = 0.90, el motor Iridium lanzará un Ir-Error: Inmutable violation.

4. Tipos Especiales (IA Ready)

Iridium incluye tipos de datos nativos para el manejo científico:

  • tensor: Matrices multidimensionales optimizadas para GPU.
  • dataset: Estructuras de datos preparadas para limpieza y análisis.
  • cronos: Variables temporales (ej: sensores que actualizan cada N segundos).
